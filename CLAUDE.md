# LLM-Council-Projects

A curated resource index of projects in the LLM Council ecosystem — multi-model deliberation systems inspired by Andrej Karpathy's original llm-council.

## Format Rules

- All resources live in a single `README.md` — no splitting across files.
- Top-level sections are ordered alphabetically.
- Domain-specific projects go under `## Domain-Specific` with `### <Domain>` sub-sections (also alphabetical). Entries there use `####` headers.
- A project may appear in more than one section if it genuinely fits multiple buckets — duplicate the entry and add an italicised `_Also listed under X._` line at the end of each cross-listed entry.
- Entries within each section are ordered alphabetically by project name.
- Resources are user-provided only — do not populate resources autonomously.
- Each GitHub entry includes shields.io badges for stars and last commit.
- Non-GitHub entries omit badges and use a simpler format.
